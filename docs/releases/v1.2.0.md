## Summary

This release converts FlutterAgentic into a Dart pub-workspace monorepo — one shared core package consumed by multiple apps — and updates every agent's rules and skills to match.

---

## Features

- The project is now a monorepo: a single shared package holds the reusable toolbelt, and each app lives in its own folder, so one dependency fetch at the root resolves everything and edits to the shared code are picked up live by every app
- Each app declares only the dependencies it uses directly and owns its own theme, constants, and entry point, keeping the shared package lean and generic

---

## Agent Context Improvements

- Every agent's setup checklist now understands the monorepo — it fetches dependencies and runs analysis across the whole workspace and checks each app's theme config
- The app-id skill is now available to all agents, and the release workflow builds a release APK from the correct app folder instead of the workspace root
- The roadmap now reflects the completed production app and the path ahead to store publishing
